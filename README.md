# CS-047: Two Universality Classes of Coordination Scaling Under Capacity Constraint

**Author:** Joe DeWit
**Target Journal:** Proceedings of the National Academy of Sciences (PNAS)

## Overview

This repository contains validation code and data for the paper "Two Universality Classes of Coordination Scaling Under Capacity Constraint." The paper identifies two universality classes — Class M (multiplicative/monopolistic) and Class T (tree-like/cooperative) — governing how coordination overhead scales in capacity-constrained networks, and derives scaling exponents from spectral properties of the underlying interaction graph.

## Analysis Scripts

Each script validates a specific claim from the paper. Scripts are numbered for logical ordering and can be run independently.

| Script | Validates | Paper Reference |
|--------|-----------|-----------------|
| `01_trust_network_exponents.py` | Power-law exponents for trust networks | Table 6 |
| `02_attachment_kernel.py` | Sublinear preferential attachment Pi(k) ~ k^0.77 | Section 5.3 |
| `03_spectral_concentration.py` | Eigenmode Concentration Property (ECP) | Section 2.3 |
| `04_cheeger_constant.py` | Class M bottleneck topology via Cheeger constant | SI Appendix E.1 |
| `05_citation_networks.py` | Class M classification of citation networks | Section 8 |
| `06_ecp_sensitivity.py` | ECP robustness to k-threshold selection | Section 2.3 |
| `07_ecp_timeseries.py` | ECP temporal robustness across bin sizes | Section 2.3 |
| `08_collaboration_spectral.py` | Spectral dimension of collaboration networks | Section 6 |
| `09_ecological_networks.py` | Cross-domain validation on ecological networks | Section 8 |
| `10_github_productivity.py` | GitHub team scaling (cached results) | Section 7, Figure 5 |
| `11_wbe_simulation.py` | WBE allometric scaling verification | Table 4 |
| `12_phase_transition.py` | N~50 phase transition prediction | Section 7.2, Figure 5 |

## Reproduction

### Requirements

```bash
pip install -r requirements.txt
```

### Running analyses

Each script can be run independently from the repository root:

```bash
python analysis/01_trust_network_exponents.py
python analysis/02_attachment_kernel.py
# ... etc.
```

Figures are saved to `figures/` and numerical results to `results/`.

**Note:** Script `10_github_productivity.py` requires GHTorrent data for full analysis. Pre-computed results are included in `results/github_scaling_results.json`.
