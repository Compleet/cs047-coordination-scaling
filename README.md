# Two Universality Classes of Coordination Scaling Under Capacity Constraint

**Author:** Joe DeWit
**Target Journal:** Proceedings of the National Academy of Sciences (PNAS)

## Overview

This repository contains validation code and data for the paper "Two Universality Classes of Coordination Scaling Under Capacity Constraint." The paper identifies two universality classes — Class M (multiplicative/monopolistic) and Class T (tree-like/cooperative) — governing how coordination overhead scales in capacity-constrained networks, and derives scaling exponents from spectral properties of the underlying interaction graph.

## Interpretation Guide

The GitHub scaling results (`results/github_scaling_results.json`) contain three exponents that are frequently confused:

| Field | Value | Meaning | Relation to Paper |
|-------|-------|---------|-------------------|
| `beta_output` | 1.34 | OLS slope of log(total PRs) vs log(N) | **Superlinear overall.** The paper's Class T claim does NOT come from this number. |
| `beta_overhead` | 2.08 | OLS slope of log(total comments) vs log(N) | Overhead grows faster than output. |
| `overhead_per_pr_exponent` | 0.74 | = beta_overhead − beta_output | Brooks' Law coefficient: how coordination cost per PR grows with team size. **Not a scaling exponent.** |

The paper's Class T prediction (β ≈ 0.75) comes from spectral dimension measurements on collaboration networks (CondMat, AstroPh, Angular) via the WBE mapping β = d_s/(d_s+1). See script 08 and SI Appendix I.

The GitHub data's contribution to the paper is **phase transition evidence**: variance of β peaks 56× at N ≈ 100-200 (script 15), confirming the Landau-Ginzburg prediction.

- Lognormal preferred over power-law for all trust networks
- Spectral dimension sensitivity for small networks
- Two null models producing different z-score ranges
- GitHub data dependency for scripts 10 and 15
